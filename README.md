### 🔵 CASE FILE #001 — QUICKADS

**`Jan 2025 → Present · Founding SDE`**

> **The brief:** Build an AI-native AdTech platform from a blank repo. No playbook.

**What was broken:** Marketers had zero intelligence on what was working for competitors. No scale. No real-time data. No AI layer.

**What I built:**
- 📚 **Ad Inspiration Library** — 1M+ ads indexed with quality ratings + hook analysis
- 🔍 **Live Search Engine** — real-time Meta scraping that actually returns signal, not noise
- 📊 **Eve** — analytics across 4 marketing funnels (awareness → conversion)
- 🤖 **ADAM** — autonomous agents that scrape competitor ads daily, without being asked
- 🎨 **Design System** — reusable component library that cut frontend dev time by 40%
- ⚡ **Architecture** — Next.js refactor that dropped page load time by 30%

**The real achievement:** Going from intern with a blank repo to the founding SDE who architected the entire product.

### 💹 CASE FILE #002 — STOCKTAX PRO

**`Personal Project · In Beta`**

> **The brief:** Indian traders pay too much tax because no tool does this correctly.

**The problem I kept hitting:** Three brokers, one PAN. Tax season = nightmare. None of the existing tools handle wash sales, multi-broker harvesting, or Section 234 planning.

**What I built:**
- 🏦 **Multi-Broker Unification** — AngelOne, Zerodha, Upstox all under one dashboard
- 🌾 **Tax Loss Harvesting Optimizer** — scores unrealized losses, recommends the minimum stocks to sell for maximum tax saving
- 🧠 **GPT-4o-mini Integration** — personalized harvest recommendations + wash sale flagging with plain-English explanations
- 📐 **Tax Simulation Engine** — model what-if trades before you place them; Section 234B/C penalty calculator
- 📂 **One-click Import** — SheetJS + broker CSV parsing that just works

**Stack:** `React` `TypeScript` `Zustand` `SheetJS` `GPT-4o-mini`

**Status:** Running in beta. Built because I needed it myself.
